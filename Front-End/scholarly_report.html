1. Abstract

BetR! (Bet on Rats!) is a website designed for rat-racing enthusiasts, that offers the possibility for authenticated users to view ongoing rat races, add money, make bets on rat races and win money. The website has many functionalities, like seeing your bet history, changing your profile information and check the history of certain rats.

2. Introduction

In our overview of mouse racing, we try to understand how a race works and how this industry has changed over time. To do this, we need a clear and consistent definition of what mouse racing is and how it is different from any other form of racing.
Rats races are seen as races in which mice have to run aimlessly just to get a piece of cheese or a reward, but from them, some people can get rich overnight. It all consists of the chosen mouse and the way it runs.
The key issue is that this type of betting is difficult to distinguish from other forms of betting, such as football betting, tennis betting, basketball betting. The lines between these different forms of betting are often blurred, as people only see the opportunity for enrichment, but not the fact that mice are kept hungry in order to run after cheese.
Here, we have created a portal where each player can bet on any mouse he wants and can see how he did in the last matches to see what the chances of winning are.
The following sections outline the application requirements, features, how our users interact with it, and the responsibilities of each team member.

3. Project Description

Implement a web application offering authenticated users the possibility of making bets on rat races.

3.1 Features


The main features of the application are:
  • Authentication: Allows the users to sign up and sign in to use the website
  • Making a bet: Allows a user with an account to place a bet on certain races
  • Modifying the profile: The user can change their username, avatar, email and any other info
  • Seeing a bet history: Users can see a history of their bets and they can check if the bets were successful or not. They can also download their bet history in HTML/CSV/JSON formats.
  • Checking a rat profile: Users can see the status of a rat's last five races, as well as information about that rat.
  • Adding a match : Admins can add matches for normal users to view and bet on.

3.2 User Interactions

  1. When the user first enters the site, they will be redirected on the Landing Page.
  2. After that, the user can choose between the Login option (if they already have an account) or the SignUp option (if they want to register). According to their choice, they will be taken on the specific page.
  3. Once they are logged in, they will be on the Dashboard Page, where they will have a Menu with different option:
    • Dashboard
    • User Profile Settings
    • Your Bets
    • Add Money
    • Add a match (Admin only)
    • Add a rat (Admin only)
  4. On the Dashboard Page, the user can see general information about how many matches are today and in the future and place a bet if he want, he can see how much money he has.
    All of the matches have two rats. Each user can place a bet consisting of one or multiple matches (on the same ticket). The elevation is calculated by adding each elevation the user pressed on. The possible profit is calculated by multiplying the amount of money the user wants to bet with the total elevation. If the user places his bet, it will apear on the Your Bets page and the winner will be chosen at random after the match is over (15 minutes after starting time).
  5. On the Past Matches Page, the user can choose a date and have all the matches from that day, with their outcomes, displayed.
  6. On the User Profile Settings Page, the user can change his personal information.
  7. On the Your Bets Page, the user can see his tickets and if he won or not. If a ticket was guessed correctly (i.e. all the matches on the ticket were guessed correctly), funds will be added automatically to the user's account when accessing this page.
  8. On the Add a match Page, the admin can add a match or delete it.
  9. On the Rat Profile Page, the user can see all information about one specific rat.
  10. On the Add money Page, the user can add money (valid credit card number is required as a regex is used)

4. Technologies used

4.1 HTML5 / CSS3 / JavaScript

  • HTML5: The standard markup language for displaying documents in a browser. It is the backbone of the front-end.
  • CSS3: The style sheet language used for describing the presentation of a document written in HTML. Heavily used in styiling our website.
  • JavaScript: Mainly used for placing bets. The user clicks the buttons which have the elevations written on them and, using JS, we can deactivate or reactivate buttons and modify the final sum according to the buttons we pressed.

4.2 PHP

PHP is a general-purpose, server side scripting language, suited for web development, that can be embedded in HTML. It is used mainly for managing a database, to track sessions and to do verifications which wouldn't be possible without scripting in general. In our application it is used often to query the database to print on the screen certain things, like the current user's bets, all of today's matches or the current user's information. It is also used to check the user's role, to redirect users which are not logged in, or to generate reports (HTML/CSV/JSON).

4.3 MySQL

MySQL is a very popular DBMS, which we use to manage our database and our tables. We have 5 tables in our database, used by PHP in queries.
  • Users: Keeps track of all the registered users (name, age, email and so on).
  • Rat: Keeps information related to rats, like their performance, their name, their place of birth.
  • Matches: Has information about the participants and the status of a match.
  • Tickets: Keeps information about the bets placed by the users (like the profit or the rat they betted on).
  • Matches_Tickets: The table which connects the Matches and the Tickets tables (as a ticket can have multiple matches betted on).

    • 4.4 Ajax

    Ajax is a set of web development techniques using many web technologies on the client-side to create asynchronous web applications. With Ajax, web applications can send and retrieve data from a server asynchronously without interfering with the display and behaviour of the existing page. We use Ajax for the past matches page, as we retrieve matches from a certain date without refreshing the page. (after the user chooses the date).
