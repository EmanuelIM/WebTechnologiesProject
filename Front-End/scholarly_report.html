1. Abstract

BetR! (Bet on Rats!) is a website designed for rat-racing enthusiasts, that offers the possibility for authenticated users to view ongoing rat races, add money, make bets on rat races and win money. The website has many functionalities, like seeing your bet history, changing your profile information and check the history of certain rats.

2. Introduction

In our overview of mouse racing, we try to understand how a race works and how this industry has changed over time. To do this, we need a clear and consistent definition of what mouse racing is and how it is different from any other form of racing.
Rats races are seen as races in which mice have to run aimlessly just to get a piece of cheese or a reward, but from them, some people can get rich overnight. It all consists of the chosen mouse and the way it runs.
The key issue is that this type of betting is difficult to distinguish from other forms of betting, such as football betting, tennis betting, basketball betting. The lines between these different forms of betting are often blurred, as people only see the opportunity for enrichment, but not the fact that mice are kept hungry in order to run after cheese.
Here, we have created a portal where each player can bet on any mouse he wants and can see how he did in the last matches to see what the chances of winning are.
The following sections outline the application requirements, features, how our users interact with it, and the responsibilities of each team member.

3. Project Description

Implement a web application offering authenticated users the possibility of making bets on rat races.

3.1 Features


The main features of the application are:
  • Authentication: Allows the users to sign up and sign in to use the website
  • Making a bet: Allows a user with an account to place a bet on certain races
  • Modifying the profile: The user can change their username, avatar, email and any other info
  • Seeing a bet history: Users can see a history of their bets and they can check if the bets were successful or not.
  • Checking a rat profile: Users can see the status of a rat's last five races, as well as information about that rat.
  • Adding a match (SA): Admins can add matches for normal users to view and bet on.

3.2 User Interactions

  1. When the user first enters the site, they will be redirected on the Landing Page.
  2. After that, the user can choose between the Login option (if they already have an account) or the SignUp option (if they want to register). According to their choice, they will be taken on the specific page.
  3. Once they are logged in, they will be on the Dashboard Page, where they will have a Menu with different option:
    • Dashboard
    • User Profile Settings
    • Your Bets
    • Add a match(SA Only)
    • Rat Profile
    • Add Money
  4. On the Dashboard Page, the user can see general information about how many matches are today and place a bet if he want, he can see how much money does he have.
  5. On the User Profile Settings Page, the user can change his personal informations.
  6. On the Your Bets Page, the user can see his tickets and if he won or not.
  7. On the Add a match Page, the super admin can add a match or delete it.
  8. On the Rat Profile Page, the user can see all information about one specific rat.
  9. On the Add money Page, the user can add money or withdraw.
